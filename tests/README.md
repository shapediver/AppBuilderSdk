# E2E Tests (Playwright)

Automated visual and interaction tests for AppBuilder apps.

## Folder structure

- `tests/config/scenarioActions.ts` → per-slug interaction definitions
- `tests/config/files/` → input files used by tests
- `tests/config/.app-links.json` → cached app links generated by global setup
- `tests/specs/appbuilder.spec.ts` → generated test suites per slug
- `tests/helpers/` → reusable helpers (wait, screenshot, coordinates, params)
- `tests/snapshots/` → committed baseline screenshots

## Setup

Install once:

```bash
pnpm install
```

## Running tests

```bash
SKIP_DEPLOY=1 pnpm test-e2e              # run all tests (skip deploy)
SKIP_DEPLOY=1 pnpm test-e2e-ui           # open Playwright UI
SKIP_DEPLOY=1 pnpm test-e2e --grep smoke # run only smoke tests
pnpm test-e2e-update-snapshots           # update baseline screenshots
```

`SKIP_DEPLOY=1` skips the branch creation and deploy step in global setup.
Without it, the suite deploys the current commit to the `testing` branch first.

## Adding a test

Open `tests/config/scenarioActions.ts` and add an entry:

```ts
{
  slug: "my-app-slug",
  params: { modelStateId: "abc123" }, // optional URL params
  actions: async (page, slug) => {
    await waitForModelRecomputed(page, async () => {
      await page.getByRole("button", {name: "Update"}).click();
    });
    await takeSnapshot(page, `${slug}-updated`);
  },
}
```

Each slug gets:
- **smoke** — page loads, canvas visible, no JS errors
- **visual** — full-page screenshot matches baseline
- **interaction** — runs `actions` (only when defined)

Slugs not listed in `scenarioActions.ts` still get smoke + visual tests automatically (discovered from the GrasshopperExampleModels markdown).

## Helpers

| Helper | Purpose |
|--------|---------|
| `waitForAppReady(page)` | Wait until the app is fully loaded |
| `waitForModelRecomputed(page, action)` | Run an action, wait for recompute |
| `takeSnapshot(page, name)` | Screenshot + auto-create baseline if missing |
| `getParameterElement(page, name)` | Find a parameter by its label |
| `viewportCoords(page, x, y)` | Normalized (0–1) → pixel coordinates |
| `applyUrlParams(url, params)` | Append query params to a URL |
| `rewriteToTestBranch(url, branch)` | Rewrite `/latest/` → `/branch/` |

## Picking 3D scene coordinates

```bash
pnpm pick-coords <slug>
```

Opens the app, lets you click the 3D scene, and prints normalized coordinates.

## Env vars

- `TEST_BRANCH` — branch/version to test against (default: `testing`)
- `SKIP_DEPLOY` — set to `1` to skip deploy
- `APPBUILDER_BASE_URL` — override the base URL entirely
